# KnowBase — 知识库问答助手

## 1. 项目概述

一个基于 LangChain + LangGraph 的知识库问答 Web 应用。用户向知识库提问，系统通过 LangGraph 编排多步流程（查询改写 → 检索 → 生成 → 幻觉检测），由 LLM 生成带来源引用的回答。支持固定的内置知识库和动态上传文档。

## 2. 技术栈

| 层次 | 选型 | 说明 |
|------|------|------|
| 框架 | LangChain | 文档处理、检索链、LLM 调用 |
| 编排 | LangGraph | 核心流程引擎：查询改写→检索→生成→质量检查 | P0 |
| Web 界面 | Streamlit | 快速搭建交互页面 | P0 |
| Web 界面 | Streamlit | 快速搭建交互页面 |
| 向量数据库 | Chroma | 本地存储，零配置 |
| 嵌入模型 | 硅基流动 API（BAAI/bge-m3） | 在线 API，无需本地 GPU |
| LLM | 硅基流动 / DeepSeek API | 在线调用 |
| 文档解析 | LangChain 文档加载器 | PDF/TXT/Markdown |

## 3. 功能需求

### 3.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 知识库问答 | 用户输入问题，检索知识库后生成回答 | P0 |
| 来源引用 | 回答中标注引用的文档片段和来源文件名 | P0 |
| 多轮对话 | 保留对话历史，支持追问 | P1 |
| 预设知识库 | 内置一批示例文档，开箱即用 | P0 |
| LangGraph 质量检查 | 生成回答后进行质量评估，低质量自动重试 | P1 |

### 3.2 LangGraph 工作流

LangGraph 编排核心问答流程，包含以下节点：

| 节点 | 功能 | 描述 |
|------|------|------|
| 查询改写 | 结合对话历史重写用户问题 | 修正指代、补全上下文，提高检索命中率 |
| 混合检索 | 向量检索 + BM25 同步进行 | 语义相似度 + 关键词精确匹配互补 |
| 重排序 | 对检索结果进行交叉编码重排 | 提升 Top-K 相关性 |
| 生成回答 | LLM 结合上下文生成答案 | 带来源引用，标注引用文档片段 |
| 质量检查 | 检查回答质量 | 可选：检测幻觉、缺失关键信息时触发重试 |

### 3.3 知识库管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 内置文档加载 | 启动时自动加载 data/ 目录下的预设文档 | P0 |
| 动态上传文档 | 用户通过界面上传 PDF/TXT/MD 文件，实时处理并入知识库 | P1 |
| 知识库查看 | 列出当前知识库中的所有文档 | P2 |

### 3.3 检索增强

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 向量检索 | 基于语义相似度的文档检索 | P0 |
| 混合检索 | 向量检索 + BM25 关键词检索 | P1 |
| 重排序 | 对检索结果进行重排，提高相关性 | P2 |
| 查询改写 | 对用户问题进行改写，提高检索命中率 | P2 |

## 4. 数据处理流程

```
用户上传/预设文档
    ↓
文档加载（PDF/TXT/MD → Document 对象）
    ↓
文本分割（按 chunk_size=500, overlap=100）
    ↓
嵌入 + 存储（Chroma 向量库）
    ↓
用户提问
    ↓
【LangGraph 流程编排】
    ├── 节点1: 查询改写（根据对话历史优化问题）
    ├── 节点2: 检索（向量检索 + BM25 混合检索）
    ├── 节点3: 重排序（提高结果相关性）
    ├── 节点4: LLM 生成回答（带上下文+来源）
    └── 节点5: 质量检查（可选重试）
    ↓
返回结果
```

## 6. 非功能需求

- 检索响应时间 < 5 秒（含 LLM 生成）
- 支持中英文文档
- 环境配置尽可能简单（pip install 即可运行）
- 演示数据不依赖外部网络下载（内置 data/ 目录）

## 7. 未来扩展（可选）

- 支持更多文件格式（Word、HTML、代码文件）
- 知识库文档删除/管理
- 对话记录持久化
- 接入不同 LLM 的对比模式

## 8. 环境要求

- Python 3.10+
- 操作系统无关（Windows/Linux/macOS）
- 无需 GPU，无需本地模型
- 需要注册硅基流动（或其他 API 提供商）获取 API Key
