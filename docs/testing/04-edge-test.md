# P2 边缘测试文档

## 1. 概述

**目标**: 验证系统在边界条件、异常输入和极端场景下的行为是否符合预期，确保系统鲁棒性。

**测试范围**:
- 字符串边界（空/超长/特殊字符）
- 数值边界（零/负数/极限值）
- 文档边界（空文件/超大文件/二进制）
- 并发边界（多线程并发访问）
- 网络边界（超时/不可达）
- 数据库边界（NULL/非法数据）
- LLM 边界（空回复/格式错误）
- 前端边界（空状态/网络错误/SSE 异常）

**前置条件**:
- 后端和前端测试环境配置完成

---

## 2. 边缘测试用例

### 2.1 字符串边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |
|------|---------|------|---------|--------|
| EDGE-STR-01 | 空字符串 question | `question=""` | `POST /api/chat/stream` 返回 422 | P0 |
| EDGE-STR-02 | 仅空格 question | `"   "` | 拒绝或 trimmed 后处理 | P1 |
| EDGE-STR-03 | 单字符 question | `"?"` | 正常处理，可能进 clarification | P1 |
| EDGE-STR-04 | 边界长度 question | 4096 字符 | 正常处理 | P0 |
| EDGE-STR-05 | 超长 question | 4097 字符 | 返回 422 | P0 |
| EDGE-STR-06 | Unicode/Emoji 内容 | `"你好🌍世界"` | 正常处理，不截断乱码 | P1 |
| EDGE-STR-07 | 纯标点符号 | `"？！!?"` | 正常处理 | P2 |
| EDGE-STR-08 | SQL 注入尝试 | `"'; DROP TABLE messages;--"` | 作为普通文本处理 | P1 |
| EDGE-STR-09 | XSS 尝试 | `"<script>alert('xss')</script>"` | 作为普通文本处理 | P1 |

### 2.2 数值边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |
|------|---------|------|---------|--------|
| EDGE-NUM-01 | `chunk_size=1` | 极小块大小 | 每 chunk 1 字符，正常切分 | P2 |
| EDGE-NUM-02 | `chunk_overlap=0` | 无重叠 | 相邻 chunk 不重叠 | P2 |
| EDGE-NUM-03 | `max_retries=0` | 不重试 | quality 不通过直接 finalize | P1 |
| EDGE-NUM-04 | `vector_candidate_k=1` | 极小候选数 | 最多返回 1 个结果 | P2 |
| EDGE-NUM-05 | 负数作为 limit/skip | chunks?limit=-1 | 返回 422 | P1 |

### 2.3 文档边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |
|------|---------|------|---------|--------|
| EDGE-DOC-01 | 空文件上传 | 0 字节 .txt | 返回 chunk_count=0 或错误 | P0 |
| EDGE-DOC-02 | 仅空白字符文件 | 全是空格/换行 | chunk 内容为空，被跳过 | P1 |
| EDGE-DOC-03 | 超大文件 | 超过 max_upload_mb | 返回 413 或校验错误 | P0 |
| EDGE-DOC-04 | 二进制文件伪装成 .txt | 含空字节 | 加载后内容可能乱码 | P2 |
| EDGE-DOC-05 | 不支持的文件格式 | `.exe` | 返回错误 | P0 |
| EDGE-DOC-06 | 文件名含特殊字符 | `"test (1).txt"` | 正常处理 | P2 |
| EDGE-DOC-07 | 超长文件名 | 255 字符文件名 | 正常截断或处理 | P2 |
| EDGE-DOC-08 | URL 指向不可达地址 | 超时/404 | 返回可读错误 | P1 |
| EDGE-DOC-09 | URL 重定向循环 | 无限 302 | 超时中断，返回错误 | P2 |

### 2.4 并发边界

### 2.5 数据库边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |

### 2.6 LLM 边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |
|------|---------|------|---------|--------|
| EDGE-LLM-01 | LLM 返回空字符串 | `""` | parse 函数返回默认值 | P1 |

| EDGE-LLM-05 | LLM 返回超长回答 | 10000+ tokens | 回答被截断或正常截断 | P2 |

### 2.7 前端边界

| 编号 | 测试用例 | 输入 | 预期行为 | 优先级 |
|------|---------|------|---------|--------|
| EDGE-FE-01 | 空对话列表 | 无对话 | 显示"暂无对话"空状态 | P0 |
| EDGE-FE-02 | 空知识库 | 无文档 | 空状态提示上传文档 | P0 |
| EDGE-FE-03 | 空指标面板 | 无日志 | 显示"暂无数据" | P0 |

| EDGE-FE-05 | API 超时 | 请求 > 30s | 超时提示 | P2 |
| EDGE-FE-06 | SSE 中途断流 | 流式中断 | 恢复或提示错误 | P1 |
| EDGE-FE-07 | 粘贴超长文本到输入框 | 1MB 文本 | 输入框正常显示（可能截断） | P2 |
| EDGE-FE-08 | 快速双击发送按钮 | 两次 sendMessage | 阻止第二次发送 | P1 |

---

## 3. 通过标准

- 所有 P0 边缘用例 100% 通过
- 系统不应在极端输入下崩溃（SIGSEGV/UAE）
- 错误信息应对用户友好，不暴露内部堆栈
- 前端在空 state 下应显示空状态提示而非白屏

## 4. 执行方式

```bash
cd backend && uv run python -m unittest tests.test_edge_cases -v
```
