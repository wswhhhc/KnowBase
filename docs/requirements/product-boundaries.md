# KnowBase 需求基线

## 1. 目标

KnowBase 当前版本的目标是提供一套可持续维护、可内网自托管的团队 RAG 应用骨架，而不是单纯的检索 demo。仓库应同时满足：

- 本地文档问答与来源追踪
- 单组织账号密码登录、固定角色和工作区授权
- Postgres 业务数据入口、Redis RQ 后台任务与 Chroma 本地向量库
- 前后端分离的真实产品界面
- 可验证的 API 契约与类型同步
- 明确的当前范围与限制

## 2. 当前范围

### 2.1 用户能力

- 上传本地文件或导入公开 URL
- 在应用内发起 SSE 流式问答
- 查看引用来源、知识库片段和调试检索结果
- 管理工作区、对话、书签和运行时设置；管理能力受 `admin` / `editor` / `viewer` 固定角色约束
- 在指标面板查看本地查询日志汇总

### 2.2 工程能力

- 后端通过 FastAPI 提供版本化 OpenAPI 快照
- 前端通过 `openapi-typescript` 消费后端契约
- 本地与 CI 都能执行后端测试、前端测试和前端构建
- 契约漂移可以被测试或 CI 阻断

## 3. 工作区语义

工作区的定义必须与当前实现一致：

- 工作区用于组织对话、书签以及知识库导入/查询的 `workspace_id` 作用域
- JWT 用户访问工作区资源前必须通过成员角色校验；legacy API Key 仅作为开发兼容路径保留
- 它不是多组织租户体系，也不是 SaaS 级别的组织隔离边界
- 删除工作区时，对话和书签会回落到默认工作区
- 已导入知识库数据的迁移或清理仍需要显式处理，当前不承诺自动生命周期治理

任何仓库文档都不得把当前工作区能力描述成“完整多组织租户隔离”。

## 4. 功能要求

### 4.1 对话与回答

- 系统必须通过 SSE 提供流式回答
- 在存在证据时，回答必须返回来源元数据
- 系统必须支持多轮对话
- 系统必须保留 `fast`、`balanced`、`high_quality`、`deep` 四种检索策略
- 质量检查失败时，系统应支持重试或兜底

### 4.2 知识导入与浏览

- 必须支持上传受支持的本地文档
- 必须支持导入公开 URL
- 必须支持知识库浏览、来源列表、热点片段和调试检索
- 必须支持删除来源与按工作区清空知识库

当前支持的文档格式：

- `.txt`
- `.md`
- `.pdf`
- `.docx`
- `.html`

### 4.3 组织与设置

- 必须支持工作区的增删改查
- 必须支持工作区成员授权和固定角色校验
- 必须支持对话持久化、重命名、导出和删除
- 必须支持书签保存和检索
- 必须支持运行时设置读取与更新，敏感配置仅允许 `admin` 修改且不得明文返回

### 4.4 可观测性

- 必须通过 API 暴露查询日志
- 前端必须提供指标面板
- 调试信息必须可用于定位检索、重排与质量检查链路

## 5. 契约与质量门禁

仓库必须把“契约不漂移”当作一等要求：

- `backend/openapi.json` 必须与当前 FastAPI schema 同步
- `frontend/src/shared/api/api-types.openapi.ts` 必须由当前 `backend/openapi.json` 生成
- 手写 SSE 类型必须与对应 Pydantic 模型保持同步

CI 至少要覆盖以下门禁：

- 后端测试
- 前端测试
- 前端构建
- OpenAPI / 前端类型漂移检查

## 6. 数据与版本控制策略

- 运行期数据库、向量库、日志和缓存不得提交到版本库
- `examples/` 下的演示样例与预置文档可纳入版本控制，但必须与运行期数据目录清晰区分
- `.gitignore` 需要准确表达“哪些是运行产物，哪些是版本化样例”

当前运行期持久化数据主要位于：

- `runtime/local/`
- `runtime/quickstart/`

## 7. 文档要求

仓库文档至少需要满足：

- `README.md` 只描述当前真实能力，不保留假截图
- `CONTRIBUTING.md` 包含本地验证、OpenAPI 导出和前端类型生成流程
- `docs/requirements/product-boundaries.md` 记录当前产品与工程边界
- `docs/testing/12-ci-test.md` 与实际 GitHub Actions 一致

## 8. 当前不纳入范围

- 多租户云平台
- 细粒度 RBAC
- 公网 SaaS / 托管式生产部署方案
- 分布式存储或高可用架构
- 在无外部模型提供方时仍具备完整回答能力
