# 验收测试文档

## 1. 概述

**目标**: 模拟真实用户场景，端到端验证 KnowBase 系统的完整功能流程是否符合需求规格。

**测试范围**: 从用户视角出发的完整功能链路

**前置条件**:
- 后端运行（mock LLM 或真实 API Key）
- 知识库包含测试文档（至少 2 个来源）
- Tavily API Key 已配置（用于联网搜索场景）

---

## 2. 用户场景

### 场景 1: 普通问答

| 字段 | 内容 |
|------|------|
| **描述** | 用户打开对话界面，输入一个问题，系统返回带来源的回答 |
| **前置条件** | 知识库有相关文档 |
| **步骤** | 1. 用户输入问题 → 2. 系统检索知识库 → 3. LLM 生成回答 → 4. 质量检查通过 → 5. 返回结果 |
| **预期结果** | 回答内容正确，来源引用有效，evidence 等级正确 |
| **通过标准** | SSE 收到完整事件链路，最终消息含 sources 数组 |

### 场景 2: 多轮追问

| 字段 | 内容 |
|------|------|
| **描述** | 用户在同一个对话中连续提问，第二轮问题依赖第一轮的上下文 |
| **前置条件** | 第一轮问答已完成 |
| **步骤** | 1. 问"公司年假政策" → 2. 收到回答 → 3. 追问"试用期员工适用吗" → 4. 系统改写查询 → 5. 检索+回答 |
| **预期结果** | 第二轮回答使用了查询改写（rewrite_query），结果与第一轮上下文相关 |
| **通过标准** | 上下文中包含"试用期"和"年假"相关信息 |

### 场景 3: 上传文档后问答

| 字段 | 内容 |
|------|------|
| **描述** | 用户上传一个文档，然后问文档中的内容，系统能正确检索到 |
| **前置条件** | 知识库初始为空 |
| **步骤** | 1. 上传 .md 文档 → 2. 确认 chunk_count > 0 → 3. 提问文档内容 → 4. 系统引用该来源 |
| **预期结果** | 回答正确引用新上传的文档 |
| **通过标准** | sources 数组包含该文档源 |

### 场景 4: 联网搜索兜底

| 字段 | 内容 |
|------|------|
| **描述** | 用户问的问题在知识库中找不到，系统启用联网搜索获取信息 |
| **前置条件** | Tavily 已配置，知识库无匹配内容 |
| **步骤** | 1. 提问不存在于 KB 的内容 → 2. 检索结果为空 → 3. 启用 web_search → 4. 结果并入回答 |
| **预期结果** | 回答含网络来源，`used_web_search=True` |
| **通过标准** | 答案的来源 URL 来自网络搜索 |

### 场景 5: 质量不通过重试

| 字段 | 内容 |
|------|------|
| **描述** | LLM 检查到回答质量不达标，自动扩检索并重新生成 |
| **前置条件** | 知识库有相关但不够充分的文档 |
| **步骤** | 1. 提问 → 2. 检索 → 3. 生成回答 → 4. 质量检查不通过 → 5. expand_retrieval → 6. 重新生成 → 7. 通过 |
| **预期结果** | 最终回答质量通过，`retry_count ≥ 1` |
| **通过标准** | run_query 返回 `retry_count >= 1`，最终 quality_ok=True |

### 场景 6: 对话记忆

| 字段 | 内容 |
|------|------|
| **描述** | 用户问"我刚才问了什么"，系统识别为记忆类问题 |
| **前置条件** | 同 thread 中有历史消息 |
| **步骤** | 1. 问"A问题" → 2. 问"我刚才问了什么" → 3. 路由到 answer_from_history |
| **预期结果** | 系统回答"您刚才问的是 A问题" |
| **通过标准** | `question_type` 为 `chat_memory` |

### 场景 7: 对话总结

| 字段 | 内容 |
|------|------|
| **描述** | 用户要求总结对话内容 |
| **前置条件** | 同 thread 有 3+ 轮对话 |
| **步骤** | 1. 多轮对话 → 2. 问"帮我总结一下" → 3. 路由到 summarize_history |
| **预期结果** | 返回结构化的对话摘要 |
| **通过标准** | `question_type` 为 `conversation_summary` |

### 场景 8: 模糊问题澄清

| 字段 | 内容 |
|------|------|
| **描述** | 用户发一个与知识库无关的模糊问候 |
| **前置条件** | 无特定要求 |
| **步骤** | 1. 问"你好" → 2. 路由到 clarification → 3. 返回礼貌提示 |
| **预期结果** | 系统提示用户补充具体问题 |
| **通过标准** | 回答含礼貌用语，quality_ok=True |

### 场景 9: 知识库浏览

| 字段 | 内容 |
|------|------|
| **描述** | 用户切换到知识库浏览页，查看所有文档片段 |
| **前置条件** | 知识库有多个来源的文档 |
| **步骤** | 1. 获取来源列表 → 2. 按来源过滤 → 3. 搜索关键词 → 4. 翻页 |
| **预期结果** | 来源过滤和搜索均返回正确结果，翻页正常 |
| **通过标准** | 过滤后只返回目标来源，搜索命中关键词 |

### 场景 10: 指标面板

| 字段 | 内容 |
|------|------|
| **描述** | 用户查看系统运行指标 |
| **前置条件** | 有查询日志数据 |
| **步骤** | 1. 切换 Dashboard 视图 → 2. 查看统计卡片 → 3. 切换时间范围 |
| **预期结果** | 统计数据正确，图表渲染正常 |
| **通过标准** | 统计卡片数值符合实际数据 |

### 场景 11: 删除来源后问答

| 字段 | 内容 |
|------|------|
| **描述** | 用户删除一个来源后，同关键词搜索不再返回该来源内容 |
| **前置条件** | 知识库有多个来源，其中一个包含目标关键词 |
| **步骤** | 1. 搜索关键词确认有结果 → 2. 删除该来源 → 3. 再次搜索同一关键词 → 4. 不再返回 |
| **预期结果** | 删除后搜索结果不包含已删除来源 |
| **通过标准** | 第二次搜索的 sources 不含已删除来源 |

### 场景 12: 清空知识库

| 字段 | 内容 |
|------|------|
| **描述** | 用户清空整个知识库 |
| **前置条件** | 知识库有数据 |
| **步骤** | 1. 确认 stats 非零 → 2. 执行 clear → 3. 检查 stats 归零 → 4. 搜索无结果 |
| **预期结果** | 所有数据被清除 |
| **通过标准** | `chunk_count=0, source_count=0`, `hybrid_search` 返回空 |

### 场景 13: 上传非法格式文件

| 字段 | 内容 |
|------|------|
| **描述** | 用户尝试上传不受支持的文件格式 |
| **前置条件** | 无 |
| **步骤** | 1. 选择 .exe 文件 → 2. 点击上传 |
| **预期结果** | 服务器返回错误提示 |
| **通过标准** | HTTP 422/400，提示不支持的格式 |

### 场景 14: 反馈提交

| 字段 | 内容 |
|------|------|
| **描述** | 用户对助手回答进行 👍/👎 评价 |
| **前置条件** | 对话中有 assistant 消息 |
| **步骤** | 1. 发送消息收到回答 → 2. 提交 👍 反馈 → 3. 查询消息 |
| **预期结果** | feedback 字段已更新 |
| **通过标准** | `messages[last].feedback == "👍"` |

---

## 3. 通过标准

- 所有 14 个用户场景 100% 通过
- 每个场景的真实执行结果符合预期
- 无阻塞性缺陷（S0/S1）

## 4. 执行方式

验收测试通过 `test_integration_api.py` + `test_graph.py` 中的场景级测试覆盖：

```bash
cd backend && uv run python -m unittest tests.test_integration_api tests.test_graph -v
```

前端验收需要人工在各浏览器上执行场景 1-3, 9-10。
