## 已做决策

| 决策 | 理由 |
|------|------|
| 按 Task 9 → 10 → 11 → 11B → 12 → 13 的依赖顺序推进 | 降低拆分后行为漂移和后续守卫误报风险 |
| 每个任务独立提交 | 便于审查、回滚与 CI 定位 |
| 已勾选任务仍需按验收标准重新证明 | 完成性审计要求直接证据，不能把旧勾选和历史测试结果当作充分证明 |

## 遇到的错误

| 错误 | 尝试次数 | 解决方案 |
|------|---------|---------|
| `tasks/todo.md` 显示工作区修改但无内容 diff | 1 | 保留原文件，不将其纳入本轮编辑；后续提交前再次核验 |
| 阶段 6 记录补丁两次上下文不匹配 | 2 | 停止联动补丁，分别更新规划文件并使用实际末行追加进度 |
| Definition of Done 引用路径不存在 | 1 | 不猜测路径；改用 `rg` 在技能目录中定位真实文件后再读取 |
| `wait_agent` 等待时间低于工具下限 | 1 | 改用允许的最小值 10000ms，不重复无效参数 |
| 前端审计文件读取使用了错误目录 | 1 | 使用 `rg --files` 定位实际路径，再按真实路径读取；不重复猜测目录 |
| append 测试在同一 `act` 中读取了尚未提交的 hook 状态 | 1 | 将“触发版本提示”和“选择 append”拆成两个 `act`，按用户真实交互顺序执行 |
| 后端守卫测试误用系统 Python | 1 | 改用项目 `uv run python -m pytest`，确保依赖来自锁定环境 |
